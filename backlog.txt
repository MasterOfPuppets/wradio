## Notes

- Source label mechanism (RB/SC/IC) exists in code but is hidden when only one provider returns results. Will show automatically when a second provider is active.
- Favourites concept is already covered by "sort by totalPlayTime desc" — most listened = favourites.
- Update checker currently works with `v1.4`-style tags and GitHub's `latest` release, but future hardening should fetch/compare releases by highest semantic version so release ordering quirks do not affect update detection.
- v1.5 was a major stability release fixing bugs introduced in v1.4 (Android Auto). Architecture was redesigned with clear separation of concerns.
